# Artifact for the paper *Pantomime: Constructive Leakage Proofs via Simulation*

## Introduction

This artifact reproduces Table 1 from Section 6 of the paper.

We verify the standard and secure versions of AIMCore using Pantomime and report
on the verification time. We also reproduce verification results from Table 1 of
the LeaVe paper to compare against---see the paper for a discussion.

### Claims from the paper supported (and not supported) by this artifact

**Claims Supported:**
- Table 1 (Section 6) verification results/times are reproduced.
- The simulator and projection code supporting proof effort claims (RQ3) can be
  inspected in `aimcore/`.

**Claims Not Supported:**
- The wolfSSL/libsodium bug detection findings (RQ6).

## Getting started guide

The artifact takes the form of a Docker image `pantomime.tar.gz`. You can load
it into Docker with this command:

```
$ docker load -i pantomime.tar.gz
```

(Depending on your system configuration, this may or may not require root
access.)

You can then run the Docker image with this command:

```
$ docker run --rm -it pantomime:latest
```

### Short run option
A partial table containing only Pantomime results can be generated by running

```
$ janet bench.janet --skip-leave
```

in the root directory of the image. This should take under 10 minutes on a
modern computer and will place a reproduction of Table 1 from Section 6 of the
paper under the `results` directory in a variety of formats (see [Output
structure](#Output-structure) for details). You can view the generated table in
the container with

```
$ bat results/table1-<timestamp>.md
```

## Output structure

Two folders will be created and populated when running this artifact:

- `data/`: will contain the raw data that the benchmarks produce in `.jdn`
format as [Janet](https://janet-lang.org) tables:
    * Pantomime data: `pantomime-<timestamp>-<benchmark options>.jdn`
    * LeaVe data: `leave-<timestamp>-<benchmark options>.jdn`

- `results/`: will contain reproductions of Table 1 from Section 6 of the paper
  in a variety of formats:
    * A markdown version: `results/table1-<timestamp>.md`
    * A LaTeX version: `results/table1-<timestamp>.tex`
    * A PDF rendering of the LaTeX version: `results/table1-<timestamp>.pdf`

### Retrieving the results from the Docker container

To copy a result file from the container for viewing, first obtain the container ID
by running

```
$ docker ps
```

You can then copy the file using `docker cp`:

```
$ docker cp <container ID>:<path to file in the container> <destination>
```

The tables will be located under `/artifact/results`; if you wish to copy the
generate PDF with Table 1, your `docker cp` command should look like this:

```
$ docker cp <container ID>:/artifact/results/table1-<timestamp>.pdf <destination>
```

## Step-by-Step instructions

> This artifact uses the [Janet](https://janet-lang.org) programming language
> for its scripts.

Run

```
$ janet bench.janet
```

in the root directory of the image to recreate Table 1 from Section 6 of the
paper. This will take a long time---around a day on a modern computer---because
of the long verification times for LeaVe. This can be ameliorated with

```
$ janet bench.janet --leave-cores Sodor-2 DarkRISCV-3
```

which will skip benchmarking the `Ibex-small` core and should take under 2 hours
on a modern computer. If you only want to verify a single core with LeaVe,
`DarkRISCV-3` is the fastest core to verify.

> You can also run the LeaVe benchmarks in parallel by passing `--threads-leave
> n` where `n` is the number of benchmarking threads:
>
> ``` $ janet bench.janet --threads-leave n ```
>
> Note that values above 3 provide no additional benefit---since `bench.janet`
> only benchmarks 3 LeaVe benchmarks, parallelism is capped at 3 (one instance
> per benchmark).

The `bench.janet` script has a number of other options; pass `--help` to see
them all.

```
$ janet bench.janet --help

Usage: janet bench.janet [--aimcore <path>] [--leave <path>] [--data <path>] [--results <path>]
                         [--runs-pantomime <n>] [--runs-leave <n>] [--threads-leave <n>]
                         [--cores-leave <cores...>] [--skip-pantomime] [--skip-leave] [--help]

  --aimcore:        Path to the aimcore directory (default: aimcore)
  --leave:          Path to the leave repo (default: leave)
  --data:           Directory to save raw benchmark data (default: data)
  --results:        Directory to save generated tables (default: results)
  --runs-pantomime: Number of runs for each leave benchmark (default: 1)
  --runs-leave:     Number of runs for each pantomime benchmark (default: 1)
  --threads-leave:  Number of threads for the leave benchmark. (default: 1)
                    Note: only one instance of each benchmark can run simultaneously.
  --cores-leave:    The cores to benchmark for the leave benchmark. (default: all)
                    Choices: DarkRISCV-3, Sodor-2, Ibex-small
  --skip-pantomime: Skip running the pantomime benchmarks
  --skip-leave:     Skip running the leave benchmarks
  --help:           Prints this usage information.
```

If more than 1 run is specified for Pantomime or LeaVe, the results in the
generated tables will be the average result across the runs.

## Reusability Guide

### Applying Pantomime to a new processor

The `pantomime/` directory is the Pantomime library as a standalone Haskell
package. To verify a new design, add `pantomime` as a dependency in your
`stack.yaml` (see `aimcore/stack.yaml` for an example) and import
`Pantomime`, `Pantomime.Base`, and `Pantomime.Clash`.

To verify the design, create a module (following `aimcore/src/Leak/PC/` as a
template) with an observation function, leakage circuit, simulator, and
projection function, then wire them together in a top-level function annotated
with `{-# ANN ... #-}`:

```hs
{-# ANN theory (P.Theory $ Base.axioms <> Clash.axioms) #-}
theory :: State -> Input -> Bool
theory =
  P.pantomime
    P.Pantomime
      { observation = obs,
        implementation = implementation,
        leakage = leak,
        simulator = sim,
        projection = proj
      }
```

Pantomime runs as a GHC plugin, so verification happens at compile time.
To check your design, run `stack build`; a failed proof causes a compilation
error.

### Extending or modifying AIMCore

The processor pipeline is in `aimcore/src/Core.hs`. Supporting types, the
instruction set, and the ISA interpreter are in `aimcore/src/Types.hs`,
`aimcore/src/Instruction.hs`, and `aimcore/src/ISA.hs`, respectively.

There is a single core implementation, parameterized by an applicative functor.
The standard and secure variants are the same core instantiated with `Identity`
and `PubSec` respectively---`PubSec` propagates secret taints and halts on
secret-dependent branches, while `Identity` does not.

The two verified variants are under `aimcore/src/Leak/`:

- `Leak/PC/`: Standard (`Identity`)
- `Leak/SecretPC/`: Secure (`PubSec`)

Each variant contains three files: `PC.hs` (the assembled circuit with Pantomime
annotations), `Leak.hs` (the leakage), and `Sim.hs` (the simulator).

After modifying the processor (or any proof component), run `stack build` from
`aimcore/` to re-check. To disable Pantomime checks for faster iteration,
comment out the `{-# ANN ... #-}` pragmas on `theory`,
`tickStateCorrespondence`, and `projectionCoherence` in the relevant `PC.hs`
module.

## Running Outside Docker

The following dependencies are required to run the artifact outside of Docker:

- Janet: for the benchmarking scripts.
- GHC 9.10.2 and Stack: to build and run Pantomime and AIMCore.
- z3: SMT solver used by Pantomime.
- Python 3 with pyyaml and lark: used by the LeaVe benchmarks.
- Yosys, Icarus Verilog, and yices: used by the LeaVe benchmarks.
- A TeX Live installation: for PDF table generation.

## Manifest

This section describes every top-level file and directory in the artifact and
its purpose.

- `aimcore`: The AIMCore processor source code, including proof components used
  in verification for the paper (e.g., the leakage and simulator circuits).

- `leave`: The LeaVe artifact (see [https://github.com/zilongwang123/LeaVe](https://github.com/zilongwang123/LeaVe)).

- `pantomime`: The Pantomime library source code.

- `bench.janet`: A Janet script that orchestrates benchmarking and table
  reproduction.

- `leave.janet`: A Janet script that orchestrates the LeaVe
  benchmarking. Generates raw benchmark data for LeaVe.

- `pantomime.janet`: A Janet script that orchestrates the Pantomime
  benchmarking. Generates raw benchmark data for Pantomime.

- `README.md`: This file.

- `table.janet`: A Janet script that generates tables (in `.md`, `.tex`, and
  `.pdf` formats) from the raw benchmarking data.

- `util.janet`: A Janet script containing utility functions used by the other
  scripts.

Further files and directories are created as part of the artifact as discussed
above.
